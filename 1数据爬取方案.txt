太好了，现在就把“要抓什么、从哪儿抓、抓到哪些字段、多久抓一次、抓取方式与合规注意点”一次性讲清楚。下面是围绕你给出的数据与变量设计（1.1–1.2）做的**可执行数据抓取清单**与**字段规范**。我只引用权威站点，并标注了每一类来源的具体入口与抓取要点，避免夸大或虚构。

# 一、政策文本（中央/国务院+省级科技厅）

## 1) 中央层：国务院政策文件库 & 政策解读

**目的**：获取“国务院文件/部门文件/政策解读”等正式文本（含正文与附件），用于五元组抽取与强度量化。
**权威入口**：

* 国务院政策文件库主页与分类页（含“国务院文件 / 部门文件”栏目与静态分页 `home_x.htm`）：页面稳定、可分页抓取。([政府网][1])
* 政策解读栏目（与正式文件一一对应）用于“意图/配套关系”抽取及训练：([政府网][2])

**抓取范围（建议主样本2009–至今）**

* “全部文件/国务院文件/部门文件”列表页 → 详情页（正文/元数据/附件）。
* “政策解读”列表页 → 详情页（对应关系字段 `related_doc_url`）。

**分页与模式**

* 典型分页：`.../home_0.htm`、`home_1.htm`…（静态）与栏目列表（HTML）。示例：部门文件 `.../zhengceku/bmwj/home_0.htm`；国务院文件 `.../zhengceku/gwywj/`（同结构）。

**建议抓取字段（中央层）**

* `title`（标题）、`pub_date`（发布日期，ISO8601）、`issuer`（发布机关）、`doc_no`（文号）、`index_no`（索引号）、`category`（国务院/部门/解读）、`status`（时效状态，如“现行”）、`source_url`、`html`（原文HTML）、`content_text`（抽取正文纯文本）、`attachments[]`（文件名/URL/类型）、`related[]`（关联文件/解读）、`region="CN-Central"`、`retrieved_at`、`sha256`（正文哈希）。
* **去重键**：`hash(title + doc_no + pub_date)`；如缺`doc_no`，用`title+pub_date`。

**抓取策略**

* 先抓列表页，解析详情链接，再抓详情页正文与附件URL。
* 对 PDF 附件统一下载，后续离线抽取文本（PDF→TXT），保留二进制以备核验。
* **节流/合规**：遵守 robots.txt；中国政府网 robots 明确部分 Disallow 路径，抓取政策栏目与静态分页属于允许范围；设置低 QPS（如 0.5–1 req/s）与退避策略。([政府网][3])

## 2) 省级层：各省（区、市）科技厅/科委“政策法规/政策文件/政策解读”

**目的**：覆盖地方层面的配套政策与实施细则，形成中央→地方的上下衔接图谱。
**权威入口与站点清单获取**：

* 以**广东省科学技术厅“政策法规-省政策法规”**栏目为模板（结构清晰、有分页），并且该页底部**列出了全国各省级科技管理部门官网链接**，可作为**站点种子清单**进行全省爬取初始化。([广东省科学技术厅][4])

**建议优先批次（样本建设→全国扩展）**

* 批次1（示范样本，经济/AI活跃度高）：**北京、上海、广东、浙江、江苏**（及深圳市科技创新局）。各站均有“政策法规/政策文件/解读/通知公告”等栏目，可按“栏目→分页→详情”通用策略抓取。示例入口：

  * 北京（市科委/中关村管理委）：栏目页（站点入口稳定）([政府网][5])
  * 上海（市科委）：政策信息栏目（站点稳定）([政府网][1])
  * 浙江（省科技厅）：政策文件/规章栏目（站点稳定）([中国政府网][6])
  * 广东（省科技厅“省政策法规”分页）([广东省科学技术厅][4])
  * 深圳（市科创局）：作为地级市示范（广东页提供直链）。([广东省科学技术厅][4])
* 批次2（扩展）：**山东、安徽、湖北、四川、重庆、湖南**等（均可从广东清单页的“各省市科技管理部门”链接出发）。([广东省科学技术厅][4])

**抓取字段（与中央层一致，新增地域编码）**

* 新增 `province_name`、`adcode_prov`（前2位行政区划码，供与统计数据对表；仅用省级 2 位/6 位，不需村级明细）。统计局对**统计用区划代码**的编制规则公开（自 2024 年起不再公开完整代码表，但省级2位长期稳定，可用于省级匹配）。([国家统计局][7])

**抓取策略与注意点**

* 站群差异大：优先抓“政策法规/政策文件/政策解读”等栏目；统一抽取“标题/文号/成文日期/发布日期/正文/附件”。
* 省级站点常用静态分页或简易 JS 翻页，优先解析 HTML。
* 少量站点附件为 `doc/docx/pdf`，统一下载并做 MIME 检查与后续解析。

# 二、产业/宏观指标（国家统计局“国家数据”门户）

**目的**：形成“省份-年份（或季度/月度）”面板指标，作为政策—行为映射与因果估计的右手边（KPI）。
**权威入口**：国家统计局“国家数据（National Data）”门户（含**简洁/高级查询与 JSON 接口 easyquery.htm**）。门户本身公开、可通过页面内部调用的接口返回 JSON。([国家数据][8])

**核心指标（省-年/季，按国家统计局口径）**

* 省级**GDP**（现价/不变价，年度/季度，按门户选择）
* **R&D经费支出**与**R&D强度（R&D/GDP）**（年度）
* **规模以上工业增加值**（月/年，按可用性）
* **高技术制造业增加值/比重**（年/季，若门户提供）
* **研发人员折合全时当量（FTE）**（年度）

> 注：以上均以**国家统计局公开口径**为准，以门户 JSON 返回的**指标编码**为准；确切编码（如 `A0101...`）在门户“查询-网络请求”中可见，学界常用该 JSON 接口进行省级面板构造（社区文章和开源项目亦有示例，但以门户为准做二次校验）。([知乎专栏][9])

**抓取方式**

* 使用 `easyquery.htm` 接口的**公开请求**（门户前端同源调用）按“数据库代码（如年度/地区）、节点/指标ID、时间范围”请求 JSON；再将返回的“时间 × 地区 × 指标值”重塑为长表（tidy）。
* **合规提醒**：尊重访问频率；接口虽公开，但并非正式开放 API 文档；因此加上**页面渲染结果复核**与**断点续抓**，避免高频访问。([国家数据][8])

**字段建议**

* `province_name`、`adcode_prov`（用于与政策合并）、`year`/`quarter`/`month`、`indicator_code`、`indicator_name_cn`、`unit`、`value`、`source_url`（对应查询页/接口） 、`retrieved_at`。

# 三、专利数据（国家知识产权局 CNIPA）

**目的**：获得**省级-时间**粒度的专利**申请/授权/有效量**等指标，用作产业创新KPI。
**权威入口**：

* **统计类月报/年报**（官方明确**按月发布《审查注册登记统计月报》**；年报/历史统计数据表亦公开），适合作为**结构化省级统计**源。([国家知识产权局][10])
* 专利公布公告（全文文献检索，不适合批量抓全量明细；若日后需要微观层面样本，可走“已注册后查询下载”的PSS系统，官方说明与免责声明见下）。([中国国家知识产权局电子出版物][11])

**抓取范围与频率**

* **月度**：《国家知识产权局审查注册登记统计月报》（发明/实用新型/外观设计授权量、PCT申请受理等，分国内外与省份维度）→ 抽取表格到“省-月-指标”长表。([国家知识产权局][12])
* **年度**：统计年报与历史汇总数据表（1985–2023年等，含**省份维度**与多项细分口径）→ 统一字段口径与行政区名称。([国家知识产权局][10])

**字段建议**

* `province_name`、`adcode_prov`、`period`（YYYY-MM/YY）、`indicator`（如 `patent_grant_invention`）、`scope`（国内/国外在华/合计）、`value`、`unit`、`source_url`、`table_name`（报告内表格名）、`retrieved_at`。

**注意与合规**

* **优先用“统计月报/年报/历史表”**等**官方已整理**数据，避免对 PSS 文献全文做高强度爬取（该系统登录、JS 渲染、免责声明严格，且定位公众检索/分析工具）。([知识产权综合服务平台][13])

# 四、时间跨度与样本构成建议

* **主样本：2009–至今**（电子政务与统计口径更稳定；月报/年报数据完备度更高）。
* **预训练/语料扩充**：可抓取**早于2009年的中央政策文献**与**CNIPA历史年报统计**用于语义模型领域适配，但**不直接纳入因果识别样本**以免口径不一致。([政府网][1])

# 五、变量与标注 Schema 对应的数据抓取映射

## 1) 政策五元组所需原子字段的抓取位置

* 〔目标/任务〕：正文段落与“政策解读”正文；
* 〔工具/手段〕：正文标题与条款（如“支持方式”“财政补助”“税收优惠”等关键字邻域）；
* 〔对象/主体〕：正文中“适用对象/申报主体/行业领域”；
* 〔地域/覆盖范围〕：省级来源站点 + 正文中地域限定；
* 〔时间/有效期与时点〕：成文/施行/截止日期、有效期条款；
* 〔强度/约束性（可选）〕：`doc_no`、发文机关级别、是否“决定/条例/办法/通知/意见”等类型信号；
* 〔配套资金/税收/用地/人才（可选数值/等级）〕：正文中具体金额/比例/额度项。

> 上述均来自**政策正文/解读** + 附件 PDF 内容抽取，无需额外来源。

## 2) 产业 KPI（面板）映射

* **地区-年份（或月/季）**：由国家统计局门户/JSON 接口与 CNIPA 月报/年报获得；
* **地区编码**：省级 2 位/6 位 `adcode_prov`（依据统计局“统计用区划代码编制规则”，仅用于省级聚合）。([国家统计局][7])

# 六、抓取频率、节流与合规模块

* **中央政策**：每日/每两日增量抓（对 `home_x.htm` 页做新链接差分）。([政府网][1])
* **省级科技厅**：每周增量抓（站点发布频率较低）。([广东省科学技术厅][4])
* **国家统计局**：月度/季度按需更新（例如月度工业/价格，季度/年度 GDP），统一走门户请求；**低 QPS + 指标级缓存**。([国家数据][8])
* **CNIPA**：**按月**抓取《统计月报》，**按年**抓取年报与历史汇总表；解析为结构化表格。([国家知识产权局][12])
* **Robots 与版权**：对每个域名检查 `robots.txt` 与声明（例如中国政府网 robots；CNIPA 统计页面为公开“数据/报告”栏目，允许浏览下载；PSS 文献检索遵循其免责声明与使用边界）。([政府网][3])

# 八、技术抓取要点与异常预案

1. **HTML 解析**：

* 统一使用 XPath/CSS 解析 **标题、日期、发布机关、文号/索引号、正文、附件**。中央/省级栏目页多为静态分页；注意中文全角/半角与日期格式兼容。

2. **PDF 附件**：

* 下载后用本地解析（如 pdfminer/pyPDF）抽取文本；如遇扫描件，再行 OCR（仅对必要样本，控制成本）。

3. **反爬/反自动化**：

* 设置 User-Agent、`If-Modified-Since`/ETag 条件请求；**QPS≤1**；指数退避；错误重试（HTTP 5xx/网络闪断）；对列队做持久化断点续抓。

4. **编码与断字**：

* 统一 `UTF-8` 存储；正文去页眉页脚与“打印版”噪音块；保留段落结构用于信息抽取。

5. **时间与时区**：

* `pub_date`/`issue_date` 均标准化为 `YYYY-MM-DD`（UTC+08:00 语义）。

6. **行政区匹配**：

* 采用**省级 2 位/6 位** `adcode_prov`，参考统计局**编制规则**；自 2024 年起不公开完整码表，但省级代码稳定不影响省级匹配。([国家统计局][7])

7. **合规与版权**：

* **仅抓取公开页面/公开报告**；遵守 robots 与版权声明；CNIPA PSS 属于检索分析系统，若使用，限于**低频人工/半自动**、保留登录与免责声明截图备查。([知识产权综合服务平台][13])

# 九、最小可运行数据清单（首轮就绪）

> 目标：两周内完成最小闭环数据仓，供抽取/标注与因果预检。

**A. 政策文本（中央）**

* 国务院政策文件库：抓取最近 **2009–至今** 的**列表第一页起**新增条目（含国务院文件、部门文件）与对应政策解读详情（取正文+附件 URL+元数据）。([政府网][1])

**B. 政策文本（省级）**

* 广东省科技厅“省政策法规”全量分页 + 详情（近三年保证完整），并从页内“各省市科技管理部门”导航中抽出**全国省级科技厅入口清单**（不立即抓全量，仅建站点表）。([广东省科学技术厅][4])

**C. 统计（国家统计局）**

* 省级年度：**GDP（现价）**、**R&D经费支出**、**R&D强度**、**研发人员FTE**（如门户有）；
* 季度（可选）：GDP（季）；
* 月度（可选）：规模以上工业增加值；
  以上通过 `easyquery.htm` 的**前端同源 JSON**抓取，保存**原始 JSON + 规范化长表**。([国家数据][8])

**D. 专利（CNIPA）**

* 2020–至今《**审查注册登记统计月报**》：发明/实用新型/外观设计**授权量**与 PCT 受理（省级维度）；
* 近年《**统计年报**》与**历史汇总表**：补齐 2009–2019 断点；
  输出统一长表（省/期/指标/值/单位/来源）。([国家知识产权局][12])

# 十、产出与质检

* **原始层**：HTML、PDF、JSON 原样存储（带 `source_url` 与抓取时间戳）；
* **标准层**：结构化 CSV/Parquet：

  * `policy_docs.parquet`（中央+广东示范）；
  * `nbs_panel.parquet`（省×期×指标）；
  * `cnipa_panel.parquet`（省×期×指标）；
* **对齐表**：`province_codes.csv`（省名→adcode_prov，对照并记录来源说明）。
* **质检**：

  * 政策：同日新增的中央/广东样本数与人工 spot-check（10 条）一致；
  * 统计：门户查询页人工核对 3 个指标×3 省×3 年；
  * CNIPA：随机抽取 2 个月份与官网表格核对合计值。([国家数据][8])
